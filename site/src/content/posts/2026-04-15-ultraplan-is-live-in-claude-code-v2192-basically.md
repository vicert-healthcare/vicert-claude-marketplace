`/ultraplan` is live in claude-code v2.1.92:
Basically, instead of planning stuff in your terminal, you kick it off from the CLI and it hands the whole thing off to a Claude Code web session. Your terminal stays free while the plan gets drafted in the cloud, which is kind of nice??
You can launch it three ways — the /ultraplan command, drop the word "ultraplan" somewhere in a prompt, or pick it from the approval dialog after a local plan finishes. Once it's done, you review the plan in the browser. You can highlight specific sections and comment on them, react with emojis if you don't feel like explaining stuff, and jump around using the outline sidebar.
When you're done *planning* — approve it to run right there in the browser (it'll open a PR when it's done), or get it back to the terminal to run locally. If you go the terminal route, you can either continue your current conversation with the plan injected, start fresh with just the plan as context, or just save it to a file for later.
Note — it needs a Claude Code web account and a GitHub repo ofc.
In my opinion they just rolled this out because it was leaked with the CLI code. Looking in the feature itself I cannot think of a situation where I ***can't just open another terminal window*** and do stuff there while planning is done in the "old" one.

So my conclusion: try it of course; but I am 1) not impressed 2) slightly disappointed.
Happy prompting!
